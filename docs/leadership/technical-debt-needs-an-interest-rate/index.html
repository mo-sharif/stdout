01 · the shape

Debt is not a synonym for code you dislike

Teams call everything debt: old code, ugly code, slow code, unfamiliar code. The useful version names the cost: slower delivery, more incidents, harder onboarding, or missed product options.

02 · the move

Quantify the drag

Attach debt work to a visible cost and a near-term opportunity. "Clean up billing" is weak. "Make pricing experiments one-day changes again" is a business case.

A debt register should read like a decision tool, not a shame list.

03 · the check

The best debt work often rides with feature work that already touches the risky area. Big cleanup seasons are harder to protect.
